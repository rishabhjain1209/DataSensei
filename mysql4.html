Inserting, Updating, and Deleting Data

In addition to querying data, you'll often need to modify the data in your MySQL database. This involves inserting new records, updating existing records, and deleting unwanted data.

Inserting Data:

INSERT INTO employees (first_name, last_name, department_id, salary)
VALUES ('John', 'Doe', 101, 60000);

Updating Data:

UPDATE employees
SET salary = 65000
WHERE employee_id = 101;

Deleting Data:

DELETE FROM employees
WHERE employee_id = 101;

Transactions and ACID Properties

Transactions are sequences of one or more SQL statements that are executed as a single unit of work. They ensure the consistency and integrity of your data by following the ACID properties:

  1. Atomicity: All changes within a transaction are treated as a single unit. If any part of the transaction fails, the entire transaction is rolled back.
  2. Consistency: A transaction brings the database from one consistent state to another. The database must satisfy all constraints after a successful transaction.
  3. Isolation: Transactions are isolated from each other until they are committed. This prevents concurrent transactions from interfering with each other.
  4. Durability: Once a transaction is committed, its changes are permanent, even in the event of system failures.

Example: Using transactions

START TRANSACTION;
UPDATE accounts SET balance = balance - 100 WHERE account_id = 101;
UPDATE accounts SET balance = balance + 100 WHERE account_id = 102;
COMMIT;

Managing Indexes for Performance Optimization

Indexes are data structures that improve the speed of data retrieval operations, especially for large tables. Common types of indexes include primary keys, unique indexes, and non-unique indexes.

Example: Creating an index on a column

CREATE INDEX idx_last_name ON employees (last_name);

Example: Dropping an index

DROP INDEX idx_last_name ON employees;

Handling NULL Values and Default Values

NULL is a special marker used in MySQL to indicate that a data value does not exist in the database. You can specify default values for columns to handle cases where no value is provided during insertion.

Example: Defining a column with a default value

CREATE TABLE students (
    student_id INT PRIMARY KEY,
    first_name VARCHAR(50) NOT NULL,
    last_name VARCHAR(50) NOT NULL,
    birth_date DATE DEFAULT '2000-01-01'
);

Example: Inserting a row with NULL and default values

INSERT INTO students (student_id, first_name, last_name)
VALUES (1, 'John', 'Doe');
