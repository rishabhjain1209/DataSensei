Module 2: SAS Programming Basics

SAS Syntax and Data Step:

SAS programming involves writing code in the SAS language to perform various data manipulation and analysis tasks. The fundamental building block of SAS programming is the DATA step. In the DATA step, you read, modify, and create datasets. Here's an overview of the basic syntax and structure of a DATA step:


DATA output_dataset;
   /* SAS code for data manipulation */
RUN;
  • DATA output_dataset;: This line defines the name of the output dataset you want to create or modify. You can specify any valid SAS dataset name.
  • /* SAS code for data manipulation */: This is where you write SAS code to perform data operations. SAS code within the DATA step includes various statements to read, process, and transform data.
  • RUN;: The RUN; statement indicates the end of the DATA step.

Creating and Modifying Datasets:

In SAS, you can create new datasets or modify existing ones using the DATA step. Common operations include:

  • Creating a New Dataset: To create a new dataset, specify its name in the DATA statement. For example:

DATA new_dataset;
   /* SAS code to create new_dataset */
RUN;
  • Modifying an Existing Dataset: You can modify an existing dataset by specifying its name in the DATA statement and using data manipulation techniques within the DATA step.

Working with Variables and Observations:

SAS datasets are organized into rows and columns, with rows representing observations and columns representing variables. Here are some common tasks related to variables and observations:

  • Selecting Variables: To include specific variables in your analysis, use the SET statement followed by the dataset name and a list of variable names. For example:

DATA selected_vars;
   SET original_dataset (keep = var1 var2);
   /* SAS code to work with selected variables */
RUN;
  • Filtering Observations: You can filter observations based on specific conditions using the IF statement. For example:

DATA filtered_obs;
   SET original_dataset;
   IF condition; /* Specify your filtering condition */
   /* SAS code to process filtered observations */
RUN;
  • Creating New Variables: To create new variables, use the DATA step to define the new variable's name and calculate its value based on existing variables.

Data Transformation and Manipulation:

SAS provides a wide range of functions and operations for data transformation and manipulation within the DATA step. Some common tasks include:

  • Data Cleaning: Identify and handle missing values, outliers, and data quality issues.
  • Data Aggregation: Summarize data using PROC MEANS, PROC SUMMARY, or other aggregation techniques.
  • Data Merging: Combine datasets using the MERGE statement or SQL joins.
  • Data Sorting: Sort data using the SORT procedure or the SORT statement within the DATA step.
  • Data Subset Selection: Create subsets of data using the WHERE statement or WHERE condition.
  • Data Transposition: Reshape data from wide to long format or vice versa using data transformation techniques.
  • Data Labeling: Assign labels to variables and values using the LABEL statement.

These fundamental concepts in SAS programming are essential for data preparation, manipulation, and analysis in various domains, including research, business analytics, and healthcare.
