Module 2: The Linux File System

Lesson 1: File System Hierarchy

In this lesson, we'll explore the Linux file system hierarchy, which is the structure that organizes files and directories on a Linux system. Understanding this hierarchy is crucial for effective file management and navigation.

File System Hierarchy:

  • In Linux, the root directory is denoted by /, and all other directories and files are organized beneath it.
  • Key directories include /bin (binary executables), /etc (system configuration files), /home (user home directories), /var (variable data), /tmp (temporary files), and /usr (user programs).
  • The /mnt directory is often used for mounting external devices like USB drives.
  • Use the ls command to list the contents of a directory and cd to change your current directory.

Code Example (List Contents of /etc Directory):

ls /etc

This command lists the contents of the /etc directory, which contains system configuration files.

Lesson 2: File and Directory Permissions

In this lesson, we'll cover file and directory permissions in Linux. Understanding permissions is vital for ensuring the security and privacy of your files and directories.

File and Directory Permissions:

  • Linux uses a permission system based on three levels: owner, group, and others.
  • Permissions are represented as read (r), write (w), and execute (x) for each level.
  • Use the chmod command to change permissions and chown to change the file owner or group.

Code Example (Change File Permissions):

chmod 644 myfile.txt

This command changes the permissions of myfile.txt to read-only for the owner and read-only for others.

Lesson 3: File Manipulation Commands

In this lesson, we'll explore essential file manipulation commands in Linux, allowing you to create, move, copy, and delete files and directories.

File Manipulation Commands:

  • touch: Create an empty file or update the access timestamp of an existing file.
  • mkdir: Create a new directory.
  • cp: Copy files or directories from one location to another.
  • mv: Move or rename files and directories.
  • rm: Remove files and directories.

Code Example (Create a New Directory and File):

mkdir myfolder
touch myfolder/myfile.txt

These commands create a new directory called myfolder and an empty file called myfile.txt within it.

Code Example (Copy and Rename a File):

cp myfile.txt myfolder/
mv myfile.txt myfolder/newfile.txt

These commands copy myfile.txt to the myfolder directory and then rename it to newfile.txt within the same directory.

Code Example (Remove a File and Directory):

rm myfolder/newfile.txt
rmdir myfolder

These commands remove the file newfile.txt and then remove the myfolder directory. Be cautious when using rm as it deletes files without confirmation.

By mastering these file manipulation commands and understanding file system permissions, you'll be well-equipped to manage and navigate the Linux file system effectively. Practice and experimentation are essential to becoming proficient in these areas.
