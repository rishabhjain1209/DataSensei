Module 3: Data Import and Export

Reading Data from External Sources:

One of the fundamental tasks in data analysis is importing data from external sources into SAS for further processing. SAS provides multiple methods and procedures for reading data from various sources, including:

  1. Reading Text Files: You can use the DATA step with the INFILE statement to read data from plain text files, such as CSV, tab-delimited, or space-delimited files. Here's an example:

DATA mydata;
   INFILE 'data.csv' DLM=','; /* Read a CSV file */
   INPUT var1 var2 var3;
RUN;
  1. Reading Excel Files: You can use the LIBNAME statement with the EXCEL engine to read data directly from Excel files (.xlsx). Alternatively, you can use the IMPORT procedure to import Excel files.
  1. Reading Databases: SAS can connect to various databases (e.g., SQL Server, Oracle, MySQL) using the appropriate SAS/ACCESS engines and procedures to read data.
  1. Reading Other Data Formats: SAS supports reading data from various other formats, such as SAS datasets, JSON, XML, and more, using the appropriate procedures and data steps.

Exporting Data to Various Formats:

SAS allows you to export datasets to different file formats for sharing, reporting, or analysis in other tools. Some common methods for exporting data include:

  1. Export to Text Files: You can use the EXPORT procedure or the DATA step with the FILE statement to export data to plain text files in various formats (e.g., CSV, tab-delimited).
  1. Export to Excel: SAS provides the EXPORT procedure to export datasets directly to Excel files (.xlsx) or other spreadsheet formats.
  1. Export to Databases: You can use the DATA step or the appropriate SAS/ACCESS engine to export data to relational databases.
  1. Export to SAS Transport Files: SAS datasets can be saved in transport file format for transferring data between different SAS environments.
  1. Export to Other Data Formats: SAS offers procedures for exporting data to other formats like JSON, XML, and more, depending on your needs.

Here's an example of exporting a SAS dataset to a CSV file:


DATA mydata;
   SET original_dataset;
   /* SAS code to process data */
RUN;
/* Export the modified dataset to a CSV file */
PROC EXPORT DATA=mydata
   OUTFILE='output.csv'
   DBMS=CSV REPLACE;
RUN;

Data Cleaning and Validation:

Before and after importing data, it's crucial to perform data cleaning and validation to ensure data quality and consistency. SAS provides tools and techniques for data cleaning and validation, including:

  • Handling Missing Data: Identify and handle missing values using functions like MISSING and techniques like imputation.
  • Data Transformation: Perform data transformations to convert data types, standardize variables, and create derived variables.
  • Data Validation: Validate data against predefined business rules or data quality standards using conditional statements and data checks.
  • Data Imputation: Impute missing values using statistical techniques or domain-specific knowledge.
  • Data Profiling: Use procedures like PROC MEANS and PROC FREQ to generate summary statistics and identify potential issues in the data.
  • Data Quality Assessment: Assess data quality and integrity by examining distributions, outliers, and relationships within the data.

Data cleaning and validation are essential steps in data preparation to ensure that the data used for analysis and reporting is accurate and reliable.

In summary, Module 3 covers the critical aspects of importing data from various sources into SAS, exporting data to different formats, and performing data cleaning and validation to ensure data quality and integrity for subsequent analysis and reporting tasks.
