Lesson 2.1: Understanding Git Workflow

Git follows a simple workflow:

  1. Working Directory: Your local project directory where you make changes to files.
  2. Staging Area: An intermediate area where you prepare changes for commit.
  3. Repository: The Git database where all commits and project history are stored.

Lesson 2.2: Basic Git Commands (init, clone, add, commit, status)

git init: Initializes a new Git repository in the current directory.

git init

git clone : Creates a local copy of a remote repository.

git clone https://github.com/username/repo.git

git add : Adds changes from the working directory to the staging area.

git add myfile.txt

git commit -m "Commit message": Commits changes in the staging area with a descriptive message.

git commit -m "Added myfile.txt"

git status: Shows the status of your working directory and staging area.

git status

Lesson 2.3: Viewing the Commit History (log)

git log: Displays the commit history of the repository.

git log

Lesson 2.4: Working with Branches (branch, checkout)

git branch: Lists all branches in the repository.

git branch

git branch : Creates a new branch.

git branch feature-branch

git checkout : Switches to a different branch.

git checkout feature-branch

Lesson 2.5: Merging Branches (merge)

git merge : Combines changes from one branch into the current branch.

git merge feature-branch

Lesson 2.6: Resolving Merge Conflicts

Merge conflicts occur when Git can't automatically merge changes from different branches. You'll need to manually resolve conflicts.

  1. Git will mark conflicts in the affected files. Open these files in a text editor.
  2. Locate the conflicting sections and decide which changes to keep.
  3. Remove conflict markers (<<<<<<<, =======, >>>>>>>) and keep the desired changes.
  4. Save the file.
  5. Add the resolved file and commit the changes:
git add resolved_file.txt
git commit -m "Resolved merge conflict"

These lessons cover the fundamental Git commands and concepts you need to work effectively with version control and collaborate with others on your projects.
