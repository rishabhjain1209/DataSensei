Lesson 4: Understanding Routes and URLs in Flask

In Lesson 4, we will delve into the fundamental concept of routing in Flask. Routing allows you to map specific URLs to functions that handle requests, making it possible to create dynamic web applications.

Routes in Flask:

  • In Flask, routes are defined using the @app.route() decorator.
  • A route maps a URL (Uniform Resource Locator) to a Python function.
  • You can use route parameters to create dynamic URLs.

Example:

        
            from flask import Flask
            app = Flask(__name__)
            @app.route('/')
            def home():
                return 'This is the home page'
            @app.route('/about')
            def about():
                return 'This is the about page'
            @app.route('/user/')
            def user_profile(username):
                return f'This is the profile page of {username}'

Lesson 5: Creating Views and Handling HTTP Methods (GET, POST)

In Lesson 5, we'll explore how to create views in Flask and handle different HTTP methods like GET and POST. Views are responsible for generating the HTML content that your users see.

Creating Views:

  • Views in Flask are Python functions associated with routes.
  • They generate HTML responses or render templates.
  • Views can also access request data, such as form submissions or URL parameters.

Example of handling GET and POST methods:

        
            from flask import Flask, request, render_template
            app = Flask(__name__)

  • For POST requests, it handles form data and performs login logic.

  1. Create a Flask application in a Python file (e.g., app.py).
  2. Define routes for different sections of your web page.
  3. Create views for each route to generate HTML content.
