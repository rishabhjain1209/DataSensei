Module 4: User and Group Management

Lesson 1: User Accounts

In this lesson, we'll cover the basics of user accounts in Linux, which are essential for managing access and permissions on a system.

User Accounts:

  • User accounts are individual identities that can log in and interact with a Linux system.
  • Each user has a unique username and associated user ID (UID).
  • User accounts are typically associated with home directories where users can store their files and configurations.

No specific code example is needed for this lesson as it provides an overview of user accounts.

Lesson 2: User Management Commands (useradd, userdel, passwd)

In this lesson, we'll explore commands for managing user accounts, including creating, deleting, and modifying user attributes.

User Management Commands:

  • useradd: Create a new user account.
  • userdel: Delete a user account.
  • passwd: Change a user's password.

Code Example (Creating a User):

sudo useradd newuser

This command creates a new user account named newuser. You may need administrative privileges to use useradd.

Code Example (Deleting a User):

sudo userdel newuser

This command deletes the user account newuser. Be cautious when using userdel as it permanently removes the user's home directory and files by default.

Code Example (Changing Password):

sudo passwd username

This command allows the user username to change their password. Replace username with the actual username.

Lesson 3: Group Management Commands (groupadd, groupdel)

In this lesson, we'll explore commands for managing groups, which allow you to organize users and assign permissions more efficiently.

Group Management Commands:

  • groupadd: Create a new group.
  • groupdel: Delete a group.

Code Example (Creating a Group):

sudo groupadd mygroup

This command creates a new group named mygroup. You may need administrative privileges to use groupadd.

Code Example (Deleting a Group):

sudo groupdel mygroup

This command deletes the group mygroup. Be cautious when using groupdel as it does not automatically remove users from the group.

Lesson 4: Permissions and Ownership

In this lesson, we'll delve into the concepts of file and directory permissions and ownership, which are crucial for controlling access to files and directories in Linux.

Permissions and Ownership:

  • Linux uses three levels of permissions: owner, group, and others, each with read (r), write (w), and execute (x) permissions.
  • The chown command changes the owner of a file or directory.
  • The chmod command modifies permissions.

Code Example (Changing Ownership):

sudo chown newowner myfile.txt

This command changes the owner of myfile.txt to newowner. You may need administrative privileges to use chown.

Code Example (Changing Permissions):

chmod 644 myfile.txt

This command changes the permissions of myfile.txt to read-only for the owner and read-only for others.

Understanding user and group management, along with permissions and ownership, is crucial for securing your Linux system and controlling access to files and directories. Practice and careful management of user accounts, groups, and permissions are key to effective system administration.
