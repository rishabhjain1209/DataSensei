Lesson 4.1: Forking Repositories

Forking a repository on GitHub allows you to create a personal copy of someone else's repository. This is useful when you want to contribute to an open-source project:

  1. Go to the repository you want to fork on GitHub.
  2. Click the "Fork" button in the top-right corner.

Lesson 4.2: Creating Pull Requests

Pull requests (PRs) are a way to propose changes to a repository. After forking and making changes to your forked repository:

  1. Go to your forked repository on GitHub.
  2. Click the "New Pull Request" button.
  3. Choose the base repository and branch where you want to merge your changes.
  4. Review your changes and click "Create Pull Request."

Lesson 4.3: Reviewing and Approving Pull Requests

Collaborators or maintainers of a repository can review and approve pull requests:

  1. Review the changes in the pull request.
  2. Add comments or suggestions.
  3. If the changes are satisfactory, click "Approve" and "Merge" to merge the pull request.

Lesson 4.4: Resolving Pull Request Comments

When someone reviews your pull request and provides comments, you can address them by making further commits:

  1. Make changes in your local repository to address the comments.
  2. Commit those changes.
  3. Push the changes to your forked repository on GitHub.
  4. The pull request will automatically update with your new commits.

Lesson 4.5: Collaborative Workflow on GitHub

Collaborative workflows involve multiple contributors working on the same repository:

  1. Clone the repository you want to contribute to:
    2. git clone
  2. Create a new branch for your changes:
    3. git checkout -b my-feature-branch
  3. Make changes, commit them, and push to your forked repository.
  4. Create a pull request from your branch to the original repository.
  5. Collaborate with others through comments and reviews.
  6. Once your pull request is approved, it can be merged into the main project.

These lessons cover essential aspects of collaborating on GitHub, from forking repositories to creating and reviewing pull requests. Collaborative workflows are vital for open-source projects and team-based development.
