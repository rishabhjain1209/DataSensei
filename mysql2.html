MySQL Data Types

MySQL supports various data types to represent different kinds of information in your database. Understanding these data types is essential for designing efficient and accurate database schemas. Here are some common data types:

  1. Numeric Data Types:
    • INT: Represents whole numbers (e.g., -5, 0, 42).
    • FLOAT: Stores floating-point numbers with decimal places (e.g., 3.14, -0.5).
    • DOUBLE: Provides higher precision for floating-point numbers (e.g., 3.14159265359).

    2. Example: Creating a table with numeric data types

        CREATE TABLE products (
        product_id INT,
        price FLOAT,
        discount DOUBLE
    );
  2. String Data Types:
    • VARCHAR: Stores variable-length character strings (e.g., 'Hello', 'MySQL').
    • CHAR: Stores fixed-length character strings (e.g., 'OpenAI', 'Data').

    3. Example: Creating a table with string data types

        CREATE TABLE customers (
        customer_id INT,
        first_name VARCHAR(50),
        last_name VARCHAR(50)
    );
  3. Date and Time Data Types:
    • DATE: Stores dates (e.g., '2023-09-06').
    • TIME: Stores times (e.g., '14:30:00').
    • DATETIME: Combines date and time (e.g., '2023-09-06 14:30:00').

    4. Example: Creating a table with date and time data types

        CREATE TABLE orders (
        order_id INT,
        order_date DATE,
        delivery_time DATETIME
    );

Understanding Constraints

Constraints are rules that define the permissible values in a column or the relationships between tables. They help ensure data integrity and maintain consistency in your database. Here are some common constraints:

  1. PRIMARY KEY: Ensures each row in a table is uniquely identifiable.

    2. Example: Adding a primary key constraint

        CREATE TABLE students (
        student_id INT PRIMARY KEY,
        first_name VARCHAR(50),
        last_name VARCHAR(50)
    );
  2. FOREIGN KEY: Defines a relationship between tables by referencing a column in another table's primary key.

    3. Example: Adding a foreign key constraint

        CREATE TABLE orders (
        order_id INT PRIMARY KEY,
        customer_id INT,
        order_date DATE,
        FOREIGN KEY (customer_id) REFERENCES customers(customer_id)
    );
  3. UNIQUE: Ensures that values in a column are unique across all rows in the table.

    4. Example: Adding a unique constraint

        CREATE TABLE email_addresses (
        email_id INT PRIMARY KEY,
        email VARCHAR(100) UNIQUE
    );
  4. NOT NULL: Requires that a column must have a value and cannot be NULL.

    5. Example: Adding a not null constraint

        CREATE TABLE employees (
        employee_id INT PRIMARY KEY,
        first_name VARCHAR(50) NOT NULL,
        last_name VARCHAR(50) NOT NULL
    );

Creating and Altering Tables with Constraints

You can create tables with constraints as shown in the examples above. To alter an existing table, you can use the ALTER TABLE statement.

Example: Adding a new column with a NOT NULL constraint

ALTER TABLE employees
ADD department_id INT NOT NULL;

Example: Adding a new FOREIGN KEY constraint

ALTER TABLE orders
ADD FOREIGN KEY (product_id) REFERENCES products(product_id);

Data Integrity and Normalization Principles

Data integrity refers to the accuracy, consistency, and reliability of data in a database. To achieve data integrity, you should follow normalization principles. Normalization involves organizing data into structured tables to eliminate redundancy and dependency issues.

Example: Normalizing tables to achieve data integrity

Let's say you have an initial table like this:

CREATE TABLE library (
    book_id INT PRIMARY KEY,
    author VARCHAR(100),
    title VARCHAR(200),
    publication_year INT,
    genre VARCHAR(50)
);

You can normalize it into separate tables like authors, books, and genres to reduce data duplication and improve data integrity.

Remember that data types and constraints play a significant role in maintaining data integrity and ensuring the accuracy of the data stored in your MySQL database.
