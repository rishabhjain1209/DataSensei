Module 1: Introduction to Linux

Lesson 1: History of Linux

In this lesson, we will explore the history of the Linux operating system. Linux was created by Linus Torvalds in 1991 as a free and open-source alternative to Unix. It has since become a powerful and widely used operating system, running on a variety of hardware platforms. Let's take a closer look at its history.

History of Linux:

  • In 1991, Linus Torvalds started working on a Unix-like kernel as a hobby project.
  • The first public release, Linux 0.01, was made available in September 1991.
  • Over the years, Linux has gained a large and active community of developers and users.
  • Major milestones include the release of the GNU General Public License (GPL) and the formation of the Free Software Foundation (FSF).
  • Linux has evolved to support a wide range of hardware architectures, from desktops and servers to embedded systems and supercomputers.

Code Example (Linux Kernel Version):

uname -a

This command displays information about the current Linux kernel version. You can try it in your Linux terminal to see the version of the kernel you're running.

Lesson 2: Linux Distributions

In this lesson, we will discuss Linux distributions, also known as distros. A Linux distribution is a complete package of the Linux kernel, system utilities, libraries, and software applications bundled together to create a functional operating system. There are many different Linux distributions available, each with its own features and purposes.

Linux Distributions:

  • Popular Linux distributions include Ubuntu, Debian, CentOS, Fedora, and Arch Linux.
  • Distributions may vary in terms of package management systems (e.g., apt, yum, pacman), desktop environments (e.g., GNOME, KDE, XFCE), and release cycles (e.g., stable, rolling release).
  • Choosing a distribution depends on your specific needs and preferences, such as ease of use, stability, and software availability.

Code Example (List Installed Distributions):

ls /etc/*-release

This command lists the release files in the /etc directory, which contain information about the installed Linux distribution. It can help you identify the distribution you are using.

Lesson 3: Linux Installation and Virtualization

In this lesson, we will delve into the process of installing Linux and explore the concept of virtualization, which allows you to run multiple operating systems on a single physical machine simultaneously.

Linux Installation:

  • Installing Linux typically involves creating bootable media (e.g., USB drive or DVD) with a Linux distribution image and booting your computer from it.
  • Follow the installation wizard to configure settings such as disk partitions, time zone, and user accounts.
  • After installation, you can update the system and install additional software packages using package managers like apt, yum, or pacman.

Code Example (Creating a Bootable USB Drive with Ubuntu):

sudo dd if=ubuntu.iso of=/dev/sdX bs=4M status=progress && sync

Replace ubuntu.iso with the actual filename of the Ubuntu ISO image and /dev/sdX with your USB drive's device name. Be cautious when using this command, as it can erase data.

Virtualization:

  • Virtualization allows you to run multiple virtual machines (VMs) on a single physical host.
  • Popular virtualization software includes VirtualBox, VMware, and KVM.
  • You can create VMs with various operating systems, including Linux, Windows, and macOS, for development, testing, or server consolidation.

Code Example (Creating a Virtual Machine with VirtualBox):

VBoxManage createvm --name "MyVM" --ostype Ubuntu_64 --register

This command creates a new virtual machine named "MyVM" with an Ubuntu 64-bit operating system. You can then configure and start the VM using VirtualBox's graphical interface.

These lessons provide a foundation for understanding Linux, its history, distributions, installation, and virtualization. Practice and exploration are essential to mastering these topics and becoming proficient with Linux.
