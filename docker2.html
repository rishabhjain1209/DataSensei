Installing Docker on Different Platforms:

Windows:

  1. Windows 10 Pro/Enterprise:
    • Install Docker Desktop for Windows.
    • Enable virtualization in your BIOS/UEFI settings (usually found under CPU settings).
  2. Windows 10 Home:
    • Install Docker Toolbox.
    • Ensure that VirtualBox is installed as it's used by Docker Toolbox for virtualization.

macOS:

  1. macOS with Intel Chip:
    • Install Docker Desktop for Mac.
  2. macOS with Apple Silicon (M1 Chip):
    • Install Docker Desktop for Mac with Apple Silicon support.

Linux:

On Linux, Docker can be installed using your distribution's package manager. However, it's recommended to follow Docker's official installation instructions for your specific distribution as they can vary.

Docker Engine Components:

Docker Engine is the core of Docker and consists of several key components:

  1. Docker Daemon: The background service responsible for building, running, and managing containers. It listens for Docker API requests and manages container lifecycle.
  2. Docker Client (CLI): The command-line tool used to interact with the Docker Daemon. You use commands like docker build, docker run, and docker ps to manage containers.
  3. Docker Images: Read-only templates that define containers. Images are used to create containers and are typically stored in registries like Docker Hub.
  4. Docker Containers: Running instances of Docker images. Containers are isolated environments where applications run with their own filesystem, network, and processes.

Docker CLI Basics:

Here are some basic Docker CLI commands:


- docker --version: Check the installed Docker version.
- docker info: Display detailed information about your Docker installation.
- docker pull <image>: Download a Docker image from a registry (e.g., Docker Hub). For example: docker pull ubuntu:20.04.
- docker build -t <tag> <path>: Build a Docker image from a Dockerfile. For example: docker build -t my-nginx-image . (assuming you have a Dockerfile in the current directory).
- docker run -d -p <host_port>:<container_port> <image>: Run a container in detached mode, mapping ports. For example: docker run -d -p 8080:80 my-nginx-image.
- docker ps: List running containers.
- docker images: List downloaded Docker images.
- docker stop <container_id>: Stop a running container.
- docker rm <container_id>: Remove a stopped container.
- docker rmi <image>: Remove a Docker image.

Example (using commands mentioned above):


# Pull the official Ubuntu 20.04 image
docker pull ubuntu:20.04
# Build a Docker image from a Dockerfile
docker build -t my-nginx-image .
# Run a container based on the image
docker run -d -p 8080:80 my-nginx-image
# List running containers
docker ps
# Stop and remove the container
docker stop <container_id>
docker rm <container_id>

These are some of the fundamental Docker CLI commands you'll use to work with containers and images. Docker provides many more commands and options for managing containers and images, and you can explore them further in the official Docker documentation.
